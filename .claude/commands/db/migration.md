# /db/migration - Generate Database Migration

Guide for creating safe database migrations with Drizzle.

## Usage

```
/db/migration <description> [options]
```

## Options

- `--add-column=<table:column:type>` - Add column to existing table
- `--add-index=<table:columns>` - Add index
- `--add-fk=<table:column:ref_table>` - Add foreign key
- `--rename=<table:old:new>` - Rename column (generates safe migration)
- `--drop-column=<table:column>` - Drop column (dangerous)

## Common Migration Patterns

### Add Column (Safe)

```sql
-- Migration: add_phone_to_users
ALTER TABLE users ADD COLUMN phone text;
```

### Add NOT NULL Column (With Default)

```sql
-- Migration: add_role_to_users
ALTER TABLE users ADD COLUMN role text NOT NULL DEFAULT 'user';
```

### Add Index (Non-blocking)

```sql
-- Migration: add_email_index_to_users
CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_users_email ON users(email);
```

### Rename Column (Safe Strategy)

```sql
-- Migration: rename_name_to_full_name
-- Step 1: Add new column
ALTER TABLE users ADD COLUMN full_name text;

-- Step 2: Copy data
UPDATE users SET full_name = name;

-- Step 3: Add constraint if needed
ALTER TABLE users ALTER COLUMN full_name SET NOT NULL;

-- Step 4: Drop old column (in separate migration after code deploy)
-- ALTER TABLE users DROP COLUMN name;
```

### Add Foreign Key

```sql
-- Migration: add_author_to_posts
ALTER TABLE posts ADD COLUMN author_id uuid REFERENCES users(id) ON DELETE CASCADE;
CREATE INDEX idx_posts_author ON posts(author_id);
```

## Workflow

```bash
# 1. Modify schema in src/db/schema/
# 2. Generate migration
pnpm drizzle-kit generate

# 3. Review generated migration in /drizzle folder

# 4. Apply migration
pnpm drizzle-kit migrate

# 5. Verify in Drizzle Studio
pnpm drizzle-kit studio
```

## Safety Checklist

- [ ] Migration is reversible or has rollback plan
- [ ] No data loss without backup
- [ ] Large table migrations use CONCURRENTLY
- [ ] NOT NULL columns have defaults
- [ ] Foreign keys have proper ON DELETE behavior
- [ ] Indexes don't duplicate existing ones
- [ ] Tested in development first

## Dangerous Operations

These require extra care:

```sql
-- DANGEROUS: Dropping column
ALTER TABLE users DROP COLUMN old_field;
-- Mitigation: Ensure no code references it, backup data

-- DANGEROUS: Changing column type
ALTER TABLE users ALTER COLUMN age TYPE text;
-- Mitigation: Add new column, migrate data, drop old

-- DANGEROUS: Adding NOT NULL without default
ALTER TABLE users ALTER COLUMN email SET NOT NULL;
-- Mitigation: Ensure no NULL values exist first

-- DANGEROUS: Dropping table
DROP TABLE old_table;
-- Mitigation: Rename first, wait, then drop
```

## File Location

- Migrations are in `/drizzle` folder
- Auto-generated by `drizzle-kit generate`
- Named with timestamp prefix
